# Coding Rules

Before writing or modifying any code, read and apply the personal coding rules.

## Process

1. Read `/Users/mayankdharwa/Desktop/obsidian-vaults/Claude-Brain/coding-rules.md`.
2. If the file exists, internalize all rules before generating code.
3. Apply every relevant rule to the code being written. If a rule is language-specific and doesn't apply to the current task, skip it silently.
4. If existing code in scope violates a rule, flag it briefly alongside your changes (one line per violation, don't block the main task).
5. If the file doesn't exist yet, proceed without rules but remind the user they can add rules with `/rule <details>`.

## Notes

- Do not recite the rules back to the user unprompted — just apply them.
- Rules are additive to Practo coding standards, not a replacement.
